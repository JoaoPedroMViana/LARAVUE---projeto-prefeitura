Interface:

- Barra lateral na esquerda com os links para os menus
- Navbar com um hamburger para acionar a side bar
- No lado direito da barra lateral deve ter um espaço para as telas serem carregadas

---------------------------------------------

Cadastro de Pessoas: 

- Inclusão, edição, exclusão [V]
- Nome (required), data de nascimento (required), CPF (required), sexo (required), cidade, bairro, rua, número e complemento [V]
- Id com autoincremento [V]
- Data de nascimento deve ser do tipo date, no banco, inclusão e edição [V]
- Sexo deve ser do tipo lista/radio/checkbox [V]
- Na edição  o CPF deve ser bloqueado [V]
- Tela para consultar todas as pessoas cadastradas em uma tabela paginada, na tabela de pessoas precisa ter ID, nome, CPF, data de nascimento e sexo [V]
- Cada registro na tabela deve ter botões de vizualizar e excluir [V]
- A opção visualizar deve levar a um form para editar a pessoa com um botão de salvar [V]
- Na tela de consulta filtro com nome, data de nascimento, CPF e sexo [falta o de data]
- Inclusão, edição e exclusão devem ter mensagens de confirmação/erro [V]
- Na exclusão fazer um modal de confirmação de ação antes de excluir [V]

---------------------------------------------

Cadastro de Protocolo:

- Descrição (permitir pelo menos 2000), data (atual), Prazo (em dias, só números), Contribuinte (pessoa linkada) [V]
- Número de identificação autoincremental, coluna com nome número (é o id do protocolo) [V]
- Data deve ser do tipo date (no form e no banco) [V]
- O campo contruibuinte deve ser uma lista com as pessoas cadastradas no sistema, no banco de dados deve vincular o id da pessoa com o protocolo (chave estrangeira) [V]
- Inclusão, edição e exclusão [V]
- Todos os campos required [V]
- Deve possuir uma tabela paginada, nela dever ter os campos número, data, prazo e nome do demandante [V]
- Para cada registro botões de vizualizar e excluir [V]
- O visualizar deve levar até um form para editar [V]
- Na página de consulta um filtro com número, descrição, data de abertura e nome contribuinte 
- inclusão, edição e exclusão devem ter mensagens de confirmação/erro [V]
- Na exclusão um modal de confirmação da ação antes de excluir [V]

---------------------------------------------

Validações:

- Deve ser utilizado o precognition [V]
- CPF deve ser único e válido 
- O prazo entre 5 e 30 dias [V]
- Contribuintes com menos de 16 não podem ter protocolo registrado

---------------------------------------------

Anexar Documentos: 

- No cadastro de protocolo campo para anexar documentos (documentos da pessoa, fotos)
- No máximo 5 documentos por protocolo
- Arquivos permitidos: JPG, JPEG, PNG e PDFs
- O tamanho máximo de cada arquivo deve ser 3MB
- É possível excluir o documento anexado, caso seja anexado indevidamente

---------------------------------------------

Autenticação:

- Cadastro de usuários
- ID autoincremental, nome completo, E-mail, senha
